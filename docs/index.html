Key Features

‚úì Single Resilient Requests

Execute individual API calls with automatic retries, backoff, caching, and rate limiting.

‚úì Generic Function Execution

Apply the same resilience guards to any async or sync function with type-safe arguments and returns.

‚úì Batch Orchestration

Execute mixed batches of requests and functions concurrently with config cascading and shared state.

‚úì Phased Workflows

Chain phases with sequential, concurrent, or mixed execution. Use non-linear control flow (CONTINUE, SKIP, REPLAY, JUMP, TERMINATE).

‚úì Graph-Based Workflows

Define DAG-based execution with parallel groups, merge points, and conditional routing for maximum parallelism.

‚úì Config Cascading

Global defaults ‚Üí group ‚Üí phase ‚Üí branch ‚Üí item overrides. Prevent repetition while maintaining fine-grained control.

‚úì State Management

Shared buffers and common buffers for passing mutable state across phases, branches, and items without global state.

‚úì Comprehensive Hooks

Pre-execution, analysis (response / error), handlers (success / error / phase / branch), and decision hooks for full control.

‚úì Trial Mode

Dry-run workflows and simulate failures safely without side effects to test retry logic and decision hooks.

‚úì State Persistence

Persist state across retry attempts for distributed tracing, audit logs, and external state management.

‚úì Metrics Guardrails

Define min/max/tolerance thresholds for execution metrics. Automatic validation with anomaly detection, severity classification (critical/warning/info), and detailed violation reporting at all levels.

Core Modules

stableRequest

Single API call execution with automatic retry, backoff, caching, rate / concurrency limits, and circuit breaking.

stableFunction

Generic function execution with identical resilience guards. Supports argument-based caching and all resilience features.

stableApiGateway

Batch orchestration for mixed requests and functions. Supports concurrent / sequential execution with config cascading and shared state.

stableWorkflow

Phased array-based workflows. Sequential, concurrent, mixed phases with non-linear control (CONTINUE / SKIP / REPLAY / JUMP / TERMINATE).

stableWorkflowGraph

DAG-based execution. Parallel groups, merge points, conditional routing with graph validation and deterministic execution.

Built-In Resilience

Intelligent Retries

  • Retry upon analyzing response, not merely an HTTP failure!
  • Settle the final error before continuing to next request.

Retry Strategies

  • FIXED: constant wait
  • LINEAR: linearly increasing wait
  • EXPONENTIAL: exponentially increasing wait
  • Jittered backoff
  • Max wait cap

Circuit Breaker

  • CLOSED ‚Üí OPEN ‚Üí HALF_OPEN states
  • Configurable thresholds
  • Recovery timeout
  • Fail fast protection
  • Dashboard metrics

Caching

  • Request/response caching
  • Function result caching
  • TTL support
  • Cache-Control header respect
  • Hit / miss metrics

Rate & Concurrency

  • Sliding-window rate limiter
  • Semaphore concurrency control
  • Queue management
  • Per-call or global instances
  • Utilization metrics

Metrics Guardrails

  • Define thresholds (min/max/tolerance)
  • Multi-level validation (request/workflow/phase)
  • Automatic anomaly detection
  • Severity classification
  • SLA monitoring & alerting

Quick Start

import { stableRequest, RETRY_STRATEGIES } from '@emmvish/stable-request';

// Single resilient request
const result = await stableRequest({
  reqData: {
    hostname: 'api.example.com',
    path: '/users/1'
  },
  resReq: true,
  attempts: 3,
  wait: 500,
  retryStrategy: RETRY_STRATEGIES.EXPONENTIAL,
  cache: { enabled: true, ttl: 5000 },
  rateLimit: { maxRequests: 10, windowMs: 1000 },
  metricsGuardrails: {
    request: {
      totalAttempts: { max: 3 },
      totalExecutionTime: { max: 5000 }
    }
  }
});

if (result.success) {
  console.log(result.data);
  
  // Check metrics validation
  if (result.metrics?.validation && !result.metrics.validation.isValid) {
    console.warn('Metrics validation failed:', result.metrics.validation.anomalies);
  }
}

Workflow Patterns

Sequential Phases

Execute phases one after another. Perfect for dependent operations where output of one becomes input to next.

Concurrent Phases

Run multiple phases in parallel. Ideal for independent workloads to maximize throughput.

Non-Linear Workflows

Use decision hooks to dynamically control flow. SKIP, JUMP, REPLAY, or TERMINATE based on conditions.

Branched Workflows

Execute independent branches with shared state. Perfect for parallel processing with synchronization points.

Graph-Based DAGs

Explicit dependency graph. Parallel groups, merge points, and conditional routing for complex topologies.

Mixed Execution

Combine requests and functions in same phases. Mix API calls with computations seamlessly.
